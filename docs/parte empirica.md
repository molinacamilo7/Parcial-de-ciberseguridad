## Parte empírica
Información que se puede obtener utilizando herramientas de análisis

A partir de las herramientas y comandos estudiados en clase es posible recopilar una gran cantidad de información sobre un sistema o una red. Esto suele conocerse como recolección
de información o reconocimiento, y constituye una de las primeras etapas dentro de un análisis de seguridad.

Por ejemplo, mediante herramientas de escaneo de red es posible identificar la dirección IP de un servidor, los puertos que se encuentran abiertos y los servicios que se están ejecutando en el sistema.

Además, también se puede obtener información relacionada con el sistema operativo utilizado, la versión de ciertos programas instalados y posibles vulnerabilidades que podrían ser explotadas por un atacante.

Análisis paso a paso de la información obtenida

Un proceso típico de análisis podría desarrollarse de la siguiente manera.

En primer lugar, se realiza una identificación del objetivo, lo cual puede incluir la dirección IP del servidor o del equipo que se desea analizar.

Posteriormente, se ejecuta un escaneo de puertos, el cual permite identificar qué servicios están disponibles en el sistema. Por ejemplo, puertos asociados con servicios web, bases de datos o conexiones remotas.

Después de identificar los servicios activos, se puede realizar un análisis más detallado para determinar qué versiones de software se están ejecutando. Esta información es importante porque algunas versiones
pueden contener vulnerabilidades conocidas.

Finalmente, se recopila información adicional sobre la infraestructura del sistema, como la configuración de la red, los protocolos utilizados y posibles puntos débiles en la seguridad.

## Búsqueda profunda de servicios, puertos e información del sistema

Una búsqueda más profunda permite identificar elementos como:

Puertos abiertos en el sistema

Servicios activos en cada puerto

Sistema operativo del servidor

Versiones de software instaladas

Posibles vulnerabilidades conocidas

Toda esta información puede ser utilizada para comprender mejor la arquitectura del sistema y detectar posibles riesgos de seguridad.

Desde el punto de vista de la ciberseguridad, este tipo de análisis es fundamental porque permite identificar debilidades antes de que puedan ser explotadas por atacantes.
